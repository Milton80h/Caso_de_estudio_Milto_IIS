# Caso_de_estudio_Milto_IIS
**Sistema de Pedidos para Cafetería Universitaria (SPCU)**

Este proyecto consiste en una aplicación web dirigida a estudiantes, docentes y personal administrativo con el objetivo de optimizar la compra de alimentos, reducir tiempos de espera y centralizar la gestión de inventarios y pagos.

* **Requisitos y Funcionalidad:** El sistema permite el registro de usuarios, visualización de menús en tiempo real, realización de pedidos en línea y notificaciones de estado. Se definen requerimientos funcionales, como la actualización automática de stock, y no funcionales, priorizando la usabilidad, disponibilidad y seguridad de los datos.
* **Pruebas:** Se documentan pruebas unitarias y de validación para garantizar la precisión en los cálculos de precios, la correcta actualización del menú y la eficiencia en el flujo de pedidos.
* **Mantenimiento y Mejoras:** Se presenta una propuesta para aplicar cuatro tipos de mantenimiento al sistema:
    * **Correctivo:** Solución de errores en precios e interfaz.
    * **Adaptativo:** Ajustes por cambios fiscales (IVA) o tecnológicos.
    * **Perfectivo:** Mejoras en la experiencia de usuario y tiempos de respuesta.
    * **Preventivo:** Implementación de pagos en línea y análisis de riesgos.
* **Análisis de Problemas:** Se identificaron deficiencias como la falta de control sobre la cantidad de pedidos y la ausencia de información nutricional. Para solucionar la falta de métricas de calidad, se propone añadir un módulo de calificación de servicio.

**Investigación sobre Markdown**

El material incluye un análisis sobre Markdown, definiéndolo como un lenguaje de marcado ligero estándar en la industria del software.

* **Uso y Utilidad:** Se destaca su filosofía de legibilidad y su importancia para la documentación técnica (archivos README), facilitando el flujo de trabajo sin depender de interfaces gráficas complejas.
* **Relación con Git y GitHub:** Se explica su compatibilidad con sistemas de control de versiones debido a su naturaleza de texto plano y cómo GitHub extendió su funcionalidad (GitHub Flavored Markdown) para incluir tablas y resaltado de código.
* **Ventajas y Desventajas:** Se valora su curva de aprendizaje mínima y portabilidad, contrastada con limitaciones en el diseño visual estricto y la falta de estandarización universal.
